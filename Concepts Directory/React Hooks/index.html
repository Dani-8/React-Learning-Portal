Visual Syntax Breakdown

useEffect (
() => { ... }

[deps]

The Action

The logic to run (Fetch, Timer, etc).

The Trigger

Only runs when these values change.

Think of `useEffect` as a Smart Window. By default, it's closed. But you can tell it to open and sync with the world whenever something inside your room (state) changes. 

useEffect(() => {
  // 1. Open Window (Setup)
  console.log("Syncing...");

  return () => {
    // 2. Close Window (Cleanup)
    console.log("Cleaning up...");
  };

}, [weather]); // Only when weather changes!

The Component Life Path

1
The Birth (Mount)

The component appears on screen. `useEffect` runs its code for the first time.

2
The Growth (Update)

A state in your dependency array changes. React re-runs the effect to keep everything in sync.

3
The Exit (Unmount)

Component leaves the screen. The Cleanup Function runs to stop timers or close connections.

useRef: The Ghost Variable

Lab Preview

Build DOM Focus Control & Value Storage that doesn’t cause re-renders.

Open Lab

Quick Summary

Think of useRef as a memory vault that React doesn't track. It lets you "remember" data without forcing the screen to refresh.

Takes (Input)

The Initial Value

Whatever you put in the parenthesis useRef(initial) becomes the starting value of .current. Usually null for DOM or 0/"" for data.

Gives (Return)

A Locked Object

It returns a stable object that stays the same for the entire life of your component. It only has one property: current.

Return Object
{
  current: value
}

The Anatomy: A Constant Identity

Unlike regular variables that get "reset" when a component re-renders, the Ref object is created once and never dies.

The Key: Changing ".current" is silent. React doesn't watch the box.

01. The DOM Anchor

Reach out and touch physical HTML elements directly. Essential for focus, measuring size, or integrating with non-React libraries.

<input ref={myRef} />

myRef.current.focus();

02. Silent Storage

Store variables that persist across renders but do not trigger a visual update when changed. Ideal for Timers or IDs.

// Saves data, stays silent.

myRef.current = "New Value";

05

Optimization Bay

Quick Summary

Optimization hooks act as a smart memory. They tell React to reuse previous work instead of starting over whenever something changes.

useMemo

Value CACHING

If you have a function that takes a long time to run (like filtering 1,000 items), useMemo saves the final result so you don't do the work twice.

In our code example:

  • Input: The price variable
  • Logic: Calculating the 15% tax
  • Output: The stored tax value

Goal: Calculate Tax only when price changes

const [price, setPrice] = useState(100);

// Result is remembered in "tax"
const tax = useMemo(() => {
return price * 0.15; 
}, [price]);

// Returns the number value directly
return <p>Tax: {tax}</p>;

useCallback

FUNCTION CACHING

Normally, React creates a brand new function on every render. useCallback keeps the exact same function in memory.

In our code example:

  • Input: The id of the item
  • Logic: Logging the "buy" message
  • Output: The stable buy function

Goal: Stop child button from re-rendering

const [id, setId] = useState(1);

// Function instance is remembered in "buy"
const buy = useCallback(() => {
console.log("Buying " + id);
}, [id]);

// Pass the same function to child
return <Button action={buy} />;
💡

When to use?

Don't use these for simple logic. Use useMemo for data that takes >10ms to calculate. Use useCallback when passing functions to components wrapped in React.memo().
